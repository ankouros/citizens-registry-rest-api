# Υπολογιστικό Νέφος και Διαχείριση Πόρων Κατά Μήκος Νεφών (3689)

### Αυτοματοποιημένη Διαχείριση RESTful Υπηρεσίας μέσω Terraform

## Εισαγωγή

Το παρόν έργο υλοποιήθηκε στο πλαίσιο της τελικής εργασίας του μαθήματος
**«Υπολογιστικό Νέφος και Διαχείριση Πόρων Κατά Μήκος Νεφών (3689)»**
του Εθνικού και Καποδιστριακού Πανεπιστημίου Αθηνών.

Στόχος της εργασίας είναι η **αυτοματοποιημένη διαχείριση μίας RESTful υπηρεσίας** και του υποκείμενου Συστήματος Διαχείρισης Βάσεων Δεδομένων (ΣΔΒΔ), μέσω της χρήσης της έννοιας της **Υποδομής ως Κώδικα (Infrastructure as Code – IaC)** και του πλαισίου **Terraform**.

Η υλοποίηση βασίζεται σε **εικονικές μηχανές (IaaS)** και ακολουθεί πλήρως το μοντέλο διαχείρισης εφαρμογών μέσω των υποκείμενων πόρων τους, όπως αυτό παρουσιάζεται στο εκπαιδευτικό υλικό του μαθήματος.

## Περιγραφή Εφαρμογής

Η εφαρμογή αφορά μία **Spring Boot RESTful υπηρεσία**, η οποία:

- εκτελείται σε πολλαπλά στιγμιότυπα εικονικών μηχανών,
- εξυπηρετεί αιτήματα μέσω εξισορροπητή φορτίου (load balancer),
- επικοινωνεί με ένα υποκείμενο Σύστημα Διαχείρισης Βάσεων Δεδομένων (ΣΔΒΔ).

Η διαχείριση της εφαρμογής πραγματοποιείται αποκλειστικά μέσω Terraform, χωρίς χειρωνακτική παρέμβαση στους πόρους νέφους.

## Αρχιτεκτονική Διάταξη

Η αρχιτεκτονική της υποδομής ακολουθεί το πρότυπο τριών επιπέδων και περιλαμβάνει:

- **Τρία (3) στιγμιότυπα εικονικών μηχανών** για τη RESTful υπηρεσία.
- **Ένα (1) στιγμιότυπο εικονικής μηχανής** για το ΣΔΒΔ.
- **Έναν εξισορροπητή φορτίου**, ο οποίος κατανέμει τα εισερχόμενα αιτήματα στα στιγμιότυπα της RESTful υπηρεσίας.
- **Ιδιωτικό εικονικό δίκτυο (VPC)** με:
  - δημόσιο υποδίκτυο για τον εξισορροπητή φορτίου,
  - ιδιωτικό υποδίκτυο για τη RESTful υπηρεσία και το ΣΔΒΔ.
- **Κανόνες ασφάλειας (security groups)** που επιτρέπουν μόνο την απαραίτητη επικοινωνία μεταξύ των επιμέρους στοιχείων.

Η αρχιτεκτονική αυτή διασφαλίζει απομόνωση, ελεγχόμενη πρόσβαση και επεκτασιμότητα της εφαρμογής.

## Οργάνωση Έργου

Η υλοποίηση του έργου είναι οργανωμένη σε σαφώς διακριτές φάσεις, καθεμία εκ των οποίων αντιστοιχεί σε συγκεκριμένο στάδιο του κύκλου ζωής της εφαρμογής:

```text
terraform/
├── prepare/   # Φάση Προετοιμασίας
├── deploy/    # Φάση Εκτέλεσης
└── destroy/   # Φάση Διαγραφής
```

Κάθε φάση διαθέτει αυτόνομη υλοποίηση και συνοδευτικό αρχείο τεκμηρίωσης (README).

## Φάσεις Διαχείρισης Υποδομής

### Φάση Προετοιμασίας (Prepare Phase)

Κατά τη φάση αυτή δημιουργείται μία εικόνα εικονικής μηχανής (AMI) για τη RESTful υπηρεσία.

Η εικόνα περιλαμβάνει το απαραίτητο περιβάλλον εκτέλεσης (Java Runtime Environment) και χρησιμοποιείται στη συνέχεια για τη δημιουργία πολλαπλών στιγμιοτύπων της υπηρεσίας.

Περισσότερες λεπτομέρειες περιγράφονται στο:

```
terraform/prepare/README.md
```

### Φάση Εκτέλεσης (Deploy Phase)

Κατά τη φάση εκτέλεσης πραγματοποιείται η πλήρης διάταξη της εφαρμογής στο υπολογιστικό νέφος:

* Δημιουργούνται τα στιγμιότυπα της RESTful υπηρεσίας και του ΣΔΒΔ.
* Δημιουργείται και ρυθμίζεται ο εξισορροπητής φορτίου.
* Ορίζονται τα δίκτυα και οι κανόνες ασφάλειας.
* Η υποδομή είναι πλήρως αυτοματοποιημένη και αναπαραγώγιμη.

Περισσότερες λεπτομέρειες περιγράφονται στο:

```
terraform/deploy/README.md
```

### Φάση Διαγραφής (Destroy Phase)

Η φάση διαγραφής επιτρέπει την πλήρη αποδέσμευση όλων των πόρων νέφους, χρησιμοποιώντας τον ενσωματωμένο μηχανισμό του Terraform.

Η καταστροφή των πόρων πραγματοποιείται με ελεγχόμενο και ασφαλή τρόπο, λαμβάνοντας υπόψη τις μεταξύ τους εξαρτήσεις.

Περισσότερες λεπτομέρειες περιγράφονται στο:

```
terraform/destroy/README.md
```

## Εκτέλεση της Εργασίας

Η εργασία εκτελείται με τη χρήση των βασικών εντολών του Terraform:

```bash
terraform init
terraform apply
terraform destroy
```

Η εκτέλεση πραγματοποιείται τμηματικά ανά φάση, σύμφωνα με τη δομή του έργου.
